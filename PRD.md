# 项目需求

## 底部TabBar导航系统配置

1. 在`app.tsx`中实现包含4个栏目的TabBar导航系统，栏目分别为：动态、好友、消息和我的

2. 为每个TabBar栏目配置两套图标资源（选中状态和未选中状态）及对应文字标签

3. 确保TabBar在所有页面中保持一致显示，且切换时具有流畅的过渡动画

4. 现在的 pages/index.tsx 页面可以直接删除或者改成动态页面（`dynamic/index.tsx`）

## 主要页面开发

1. 动态页面（`index.tsx`）

- 空白页面即可

2. 好友页面（`index.tsx`）

- 空白页面即可

3. 消息页面（`index.tsx`）

- 空白页面即可

4. 个人页面（`index.tsx`）

- 展示用户基本信息（头像、昵称、个性签名），用户信息通过 mobx 全局状态管理

- 设计功能菜单入口：设置、帮助中心、关于我们、退出登录


## 技术实现规范

1. 组件开发：所有页面和功能模块必须使用函数组件实现，结合React Hooks管理状态和生命周期

2. 样式处理：统一使用SCSS格式编写样式文件，实现响应式设计以适配不同屏幕尺寸

3. 资源管理：使用常量集中管理所有占位图资源路径，便于后续统一替换为正式资源

4. 代码组织：遵循模块化开发原则，合理划分目录结构，确保页面结构清晰、代码可维护性高

5. 性能优化：实现页面懒加载，优化图片资源加载，确保小程序运行流畅无明显卡顿

6. 状态管理：继续使用 mobx 管理库，全局管理用户登录状态、用户信息等关键信息
