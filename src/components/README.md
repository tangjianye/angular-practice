# 组件目录

本目录用于存放项目中所有的组件，按功能和使用场景进行分类。

## 目录结构

```
components/
├── common/     # 通用基础组件
├── business/   # 业务组件
└── layout/     # 布局组件
```

## 组件分类说明

### 1. common/ - 通用基础组件

通用基础组件是不依赖特定业务逻辑的、可在多个场景复用的UI组件。

**特点：**
- 高度可复用
- 不包含业务逻辑
- 只关注UI展示
- 提供丰富的配置项

**示例组件：**
- Button - 按钮组件
- Card - 卡片组件
- Avatar - 头像组件

### 2. business/ - 业务组件

业务组件是包含特定业务逻辑的组件，通常在特定的业务场景中使用。

**特点：**
- 包含业务逻辑
- 与特定功能模块紧密相关
- 可在相关业务场景中复用

### 3. layout/ - 布局组件

布局组件用于构建页面的整体结构和布局。

**特点：**
- 负责页面或区块的布局结构
- 处理响应式设计
- 提供一致的页面框架

## 组件开发规范

### 文件结构

每个组件应遵循以下文件结构：

```
组件名/
├── index.tsx    # 组件主要逻辑
└── style.scss   # 组件样式
```

### 命名规范

- **组件目录名**: 使用 PascalCase
- **组件文件名**: `index.tsx`
- **样式文件名**: `style.scss`
- **组件类名**: 使用 `custom-组件名` 作为基础类名

### 开发要求

1. 所有组件必须使用 TypeScript 开发
2. 组件必须是函数组件
3. 为组件的 props 定义清晰的接口
4. 组件样式必须使用 SCSS
5. 组件应具有良好的可配置性
6. 避免在组件中引入过多的外部依赖
7. 组件应具有合理的默认值
8. 提供必要的文档注释

### 样式规范

1. 使用 BEM 命名规范
2. 避免使用内联样式
3. 样式文件应与组件文件放在同一目录下
4. 样式类名应具有描述性
5. 避免使用 ID 选择器

## 组件使用示例

### 导入组件

```tsx
import Button from '@/components/common/Button';
```

### 使用组件

```tsx
<Button type="primary" size="large" onClick={handleClick}>
  点击按钮
</Button>
```

## 注意事项

1. 组件开发应遵循单一职责原则
2. 复杂的组件应考虑拆分为多个简单组件
3. 组件应具有良好的性能和稳定性
4. 组件应考虑可访问性
